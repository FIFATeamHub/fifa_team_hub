# UML
---

Os diagramas presentes em nosso projeto que compõem nossa UML são:

### 1. Diagrama de Classes

O programa será orientado à objetos, logo, um diagrama de classes será necessário para: 

- Mostrar quais são as classes, seus respectivos atributos e métodos presentes no backend.
- Exibir o relacionamento entre as classes.

### 2. Diagrama de Casos de Uso

Esse diagrama é criado para demonstrar o que cada role (categoria de usuário) pode fazer no programa.
Por exemplo:

- Comissão Médica -> Fazer upload de documentos médicos
- Comissão Técnica -> Editar as táticas do time e visualizar documentos
