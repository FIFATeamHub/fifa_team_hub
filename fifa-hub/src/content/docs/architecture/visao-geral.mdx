O FIFA Team Hub é uma aplicação de três camadas clássica — SPA, API REST e armazenamento de dados/arquivos — com cada camada publicada como um serviço independente no Google Cloud.

## Por que essa separação

- **Frontend e backend são publicados e escalados independentemente** — dois serviços Cloud Run distintos, cada um com seus próprios recursos de CPU/memória e política de auto-scaling.
- **Arquivos ficam fora do banco de dados.** O PostgreSQL guarda apenas o caminho do arquivo (`storage_path`) e metadados; o binário do documento nunca trafega pela camada de aplicação além do momento do upload — downloads acontecem diretamente entre o navegador e o Cloud Storage, via link assinado.
- **A lógica de negócio fica concentrada no backend.** O frontend não decide isolamento, permissões ou validação de arquivo — ele apenas reflete o que a API retorna, o que evita duplicar (e divergir) regras de segurança entre duas camadas.
