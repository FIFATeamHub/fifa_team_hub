## ✅ Decisões e Padronizações

Duas decisões importantes foram tomadas para padronizar o desenvolvimento:

**PRs com conflito serão negados diretamente.** Todo PR deve ser aberto com a branch devidamente atualizada em relação à `develop`. Conflitos devem ser resolvidos pelo autor antes de submeter o code review.

**Variáveis devem seguir o padrão em inglês e minúsculo.** Todas as variáveis de ambiente, nomes de campos e configurações seguem estritamente esse padrão. Exemplo: `gcs_bucket_name`, evitando variações como `BUCKET` ou `nomeBucket`.
