# 📂 API — Documentos

Todos os endpoints desta seção requerem obrigatoriamente autenticação via Bearer Token JWT. Todas as rotas vivem sob o prefixo **`/api/document`** (singular).

---

## 🔄 Ciclo de vida do documento

<Mermaid code={`stateDiagram-v2
    [*] --> PENDING: POST /api/document/upload
    PENDING --> APPROVED: PATCH /review (status=APPROVED)
    PENDING --> REJECTED: PATCH /review (status=REJECTED)
    PENDING --> DELETED: DELETE (pelo próprio autor)
    APPROVED --> DELETED: DELETE (pelo próprio autor)
    REJECTED --> DELETED: DELETE (pelo próprio autor)
    DELETED --> [*]
`} />

## 📋 Modelo de Dados (Document Entity)

| Campo | Tipo | Descrição |
| :--- | :--- | :--- |
| `id` | UUID | Identificador único |
| `selection_id` | UUID | Seleção do documento (extraído do token, nunca do body) |
| `uploaded_by` | UUID | ID do utilizador que fez upload |
| `type` / `doc_type` | Enum | Tipo do documento (ver abaixo) |
| `original_name` | String | Nome original do arquivo, sanitizado (`secure_filename`) |
| `storage_path` | String | Caminho no bucket GCS ou no disco local (nunca exposto na API) |
| `status` | String | `PENDING`, `APPROVED`, `REJECTED` ou `DELETED` |
| `created_at` | DateTime | Data/hora do upload |
| `deleted_at` | DateTime | Soft delete (`null` se ativo) |
| `reviewed_by` / `reviewed_at` | UUID / DateTime | Preenchidos quando o documento é revisado |

**🏷️ Enums Disponíveis:**
- **DocType:** `CONVOCADO` \| `PASSPORT` \| `LAUDO_MEDICO` \| `RELATORIO_TATICO` \| `ESQUEMA_JOGADAS`
- **DocStatus:** `PENDING` \| `APPROVED` \| `REJECTED` \| `DELETED`

---

## 📤 POST /api/document/upload

Faz upload de documento para a seleção do utilizador autenticado.

- **Autenticação Requerida:** ✅ `Bearer <token>` — quem pode enviar qual `doc_type` segue uma matriz por perfil (ver [Upload de Documentos Sensíveis](/fifa_team_hub/features/upload-documentos/)); o perfil `ORGANIZER` não pode enviar documentos.
- **Content-Type:** `multipart/form-data`

### 📥 Parâmetros do Formulário

| Campo | Tipo | Obrigatório |
| :--- | :--- | :--- |
| `file` | Arquivo binário | ✅ |
| `doc_type` | String (Enum DocType) | ✅ |

### 📦 Ver Resposta de Sucesso (201)

```json
{
  "id": "uuid",
  "original_name": "convocacao_brasil.pdf",
  "doc_type": "CONVOCADO",
  "file_size_kb": 245,
  "status": "PENDING",
  "uploaded_by": "uuid-do-user",
  "created_at": "2026-06-09T10:00:00-03:00"
}
```

### 📊 Respostas de Erro

| Status | Motivo |
| :--- | :--- |
| `400` | Ficheiro ausente no request ou `doc_type` não informado |
| `401` | Token ausente ou inválido |
| `403` | Perfil sem permissão de upload para este `doc_type` |
| `413` | Ficheiro excede 10 MB |
| `415` | Formato de ficheiro não suportado (MIME real diferente da extensão) |
| `422` | `doc_type` não é um valor válido do enum |
| `503` | Falha ao armazenar (cota do GCS excedida ou storage indisponível) |

## 🔍 GET /api/document/

Lista documentos filtrados pelo perfil do utilizador autenticado.

- **Autenticação Requerida:** ✅ `Bearer <token>`

### 🎛️ Parâmetros de Query (Opcionais)

| Parâmetro | Tipo | Padrão | Descrição |
| :--- | :--- | :--- | :--- |
| `doc_type` | String | — | Filtrar por tipo de documento |
| `page` | Integer | 1 | Número da página |
| `per_page` | Integer | 10 | Itens por página |

> - **`TECHNICAL_STAFF` / `MEDICAL_STAFF` / `ATHELETE`:** Retorna documentos da própria seleção, restritos aos tipos permitidos ao perfil (ver [Controle de Acesso](/fifa_team_hub/features/controle-acesso/))
> - **`ORGANIZER`:** Retorna `CONVOCADO` e `PASSPORT` de **todas** as seleções
> - **`AUDITOR`:** HTTP `403` (este perfil só acessa o endpoint de auditoria, não a listagem de documentos)

### 📦 Ver Resposta de Sucesso (200)

```json
{
  "data": [
    {
      "id": "uuid",
      "original_name": "convocacao.pdf",
      "doc_type": "CONVOCADO",
      "storage_url": null,
      "status": "PENDING",
      "uploaded_by_id": "uuid-do-user",
      "selection_code": "BRA",
      "created_at": "2026-06-09T10:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 10,
    "total": 45,
    "pages": 5
  }
}
```

## 📌 GET /api/document/pending

Retorna os **tipos** de documento que o usuário autenticado ainda precisa enviar (não é uma lista de documentos, é uma lista de `doc_type` pendentes segundo a matriz de obrigatoriedade do seu perfil).

- **Autenticação Requerida:** ✅ `Bearer <token>`

### 📦 Ver Resposta de Sucesso (200)

```json
[
  { "doc_type": "PASSPORT" },
  { "doc_type": "LAUDO_MEDICO" }
]
```

---

## 📄 GET /api/document/&lt;uuid:document_id&gt;

Retorna os metadados de um documento específico, já acompanhados de uma URL de download assinada (válida por 15 minutos).

- **Autenticação Requerida:** ✅ `Bearer <token>` — RBAC idêntico ao da listagem, escopado por seleção

### 📦 Ver Resposta de Sucesso (200)

```json
{
  "id": "uuid",
  "original_name": "convocacao_brasil.pdf",
  "doc_type": "CONVOCADO",
  "status": "PENDING",
  "selection_id": "uuid-da-selecao",
  "created_at": "2026-06-09T10:00:00Z",
  "download_url": "https://storage.googleapis.com/bucket/...?X-Goog-Signature=..."
}
```

### 📊 Respostas e Códigos de Status

| Status | Descrição |
| :--- | :--- |
| `200` | Documento encontrado |
| `403` | Documento pertencente a outra seleção / tipo não visível ao perfil |
| `404` | Documento não encontrado |

---

## 📥 GET /api/document/&lt;uuid:document_id&gt;/download

Gera um link temporário e assinado para a realização do download seguro do arquivo.

- **Autenticação Requerida:** ✅ `Bearer <token>`

### 📦 Ver Resposta de Sucesso (200)

```json
{
  "url": "https://storage.googleapis.com/bucket/...?X-Goog-Signature=...",
  "expires_in_minutes": 15,
  "filename": "convocacao_brasil.pdf"
}
```

### 📊 Respostas e Códigos de Status

| Status | Descrição |
| :--- | :--- |
| `200` | URL assinada gerada (válida por 15 min) |
| `403` | Documento pertencente a outra seleção |
| `404` | Documento não encontrado |
| `410` | Arquivo não encontrado no storage (removido ou corrompido) |
| `503` | Serviço de armazenamento temporariamente indisponível |

---

## 📡 GET /api/document/&lt;uuid:document_id&gt;/stream

Endpoint **interno**, usado apenas quando `STORAGE_BACKEND=local` — faz streaming direto do arquivo em disco via Flask (`send_file`), em vez de gerar uma URL assinada do GCS. Não retorna JSON, retorna o binário do arquivo.

- **Autenticação Requerida:** ✅ `Bearer <token>`

### 📊 Respostas e Códigos de Status

| Status | Descrição |
| :--- | :--- |
| `200` | Stream do arquivo (binário) |
| `403` | Acesso negado |
| `404` | Documento não encontrado no banco |
| `410` | Arquivo físico não encontrado no disco |

---

## 📝 PATCH /api/document/&lt;uuid:document_id&gt;/review

Aprova ou rejeita um documento `PENDING`. Só documentos com status `PENDING` podem ser revisados.

- **Autenticação Requerida:** ✅ `Bearer <token>` — autorrevisão é sempre bloqueada; `PASSPORT` só pode ser revisado por `AUDITOR`; `LAUDO_MEDICO` só por `MEDICAL_STAFF` da mesma seleção do documento.

### 📦 Ver Body do Request (JSON)

```json
{
  "status": "APPROVED",
  "reason": "Documentação conferida e válida"
}
```

### 📦 Ver Resposta de Sucesso (200)

```json
{
  "message": "Documento revisado com sucesso. Novo status: APPROVED",
  "id": "uuid",
  "status": "APPROVED",
  "reviewed_by": "uuid-do-revisor",
  "reviewed_at": "2026-07-09T18:00:00-03:00"
}
```

### 📊 Respostas e Códigos de Status

| Status | Descrição |
| :--- | :--- |
| `200` | Documento revisado com sucesso |
| `400` | `status` ausente/inválido, ou documento não está `PENDING` |
| `403` | Autorrevisão bloqueada, ou perfil sem permissão para revisar este `doc_type` |
| `404` | Documento não encontrado |

---

## 🗑️ DELETE /api/document/&lt;uuid:document_id&gt;

Realiza a exclusão lógica do documento (*soft delete*) e remove o arquivo físico do storage.

- **Autenticação Requerida:** ✅ `Bearer <token>` — qualquer perfil **exceto `ORGANIZER`** pode excluir, mas **somente o próprio autor do upload**, dentro da própria seleção (outro usuário da mesma seleção não pode excluir um documento que não enviou).

### 📊 Respostas e Códigos de Status

| Status | Descrição |
| :--- | :--- |
| `204` | Documento excluído com sucesso |
| `403` | Perfil sem permissão, documento de outro autor, ou de outra seleção |
| `404` | Documento não encontrado |

---

## 🛠️ Regras de Validação Geral

> - **Tamanho Máximo:** 10 MB por arquivo
> - **Formatos Aceitos:** PDF, JPG, JPEG, PNG, DOCX
> - **Validação de Integridade:** O tipo real do arquivo é validado via MIME type através da biblioteca `python-magic` (rejeitando alterações apenas na extensão)
> - **Segurança de Infraestrutura:** O `storage_path` real nunca é exposto nas respostas da API; apenas URLs assinadas e temporárias são distribuídas
> - **Segurança de Multitenancy:** O parâmetro `selection_id` é obrigatoriamente injetado a partir do token JWT e nunca aceito via payload do body
